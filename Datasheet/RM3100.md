# RM3100 Programming Guide

## Overview

The RM3100 is a 3-axis geomagnetic sensor module built from:

- The **MagI2C** ASIC controller
- One **X/Y sensor coil**
- One **Z sensor coil**

You program the module by reading and writing registers in the MagI2C. The normal workflow is:

1. Power and wire the device correctly.
2. Optionally program the cycle-count registers.
3. Trigger either a single measurement or continuous measurement mode.
4. Wait until data is ready.
5. Read 9 bytes of raw X/Y/Z data.
6. Convert each 24-bit signed value into engineering units.

This guide is SPI-first because this repository is focused on SPI demos, but the same register map also applies to I2C.

## Wiring for SPI

Use the following hardware rules when bringing up the RM3100 over SPI:

- `I2CEN` must be pulled **LOW** for SPI mode.
- Connect `SSN`, `SCLK`, `MOSI`, and `MISO` to the host SPI peripheral.
- Connect `DRDY` to a GPIO if you want a hardware data-ready signal.
- `AVDD` and `DVDD` must be within **2.0 V to 3.6 V**.
- Keep `AVDD` and `DVDD` within **0.1 V** of each other while powered.
- `DVDD` must come up before `AVDD`, or at the same time.
- `REXT` uses a **33 kOhm** timing resistor in the typical datasheet configuration.
- Reserved pins should be tied to ground.
- `NC` pins should be left unconnected.
- SPI clock must be **1 MHz or lower**.

Important pins from the datasheet:

| Pin | Name | Notes |
| --- | --- | --- |
| 1 | `MOSI` | SPI master-out / sensor command input |
| 3 | `SSN` | Active-low chip select |
| 22 | `I2CEN` | Pull LOW for SPI |
| 23 | `DRDY` | Goes HIGH when a requested measurement is ready |
| 25 | `REXT` | External timing resistor, typically 33 kOhm |
| 27 | `SCLK` | SPI clock, 1 MHz max |
| 28 | `MISO` | SPI master-in / sensor data output |

Layout and board-level notes that matter in practice:

- Keep the MagI2C within about 10 cm of the sensor coils.
- Keep each coil's paired traces close together.
- Keep high current paths, motors, speakers, magnets, and ferrous material away from the sensor coils.

## SPI Register Protocol

The RM3100 uses an 8-bit SPI command byte:

- **Write command**: send the 7-bit register address directly.
- **Read command**: send `0x80 | reg`.

Examples:

- Write `POLL` register `0x00`: send `0x00`
- Write `CMM` register `0x01`: send `0x01`
- Read `STATUS` register `0x34`: send `0xB4`
- Read `REVID` register `0x36`: send `0xB6`
- Read measurement results starting at `MX2` register `0x24`: send `0xA4`

SPI mode details from the datasheet:

- The device supports **SPI mode 0** (`CPOL=0`, `CPHA=0`)
- The device also supports **SPI mode 3** (`CPOL=1`, `CPHA=1`)
- Data is transferred MSB first
- Once the first register is addressed, the internal register pointer auto-increments, so burst reads and writes are preferred
- `SSN` must go LOW before the transaction and stay LOW for the full transaction
- It is good practice to return `SSN` HIGH after each complete read or write

## Essential Registers

These are the minimum registers you usually need for bring-up:

| Name | Addr | Read Cmd | Purpose |
| --- | --- | --- | --- |
| `POLL` | `0x00` | `0x80` | Start a single measurement |
| `CMM` | `0x01` | `0x81` | Start or stop continuous measurement mode |
| `CCX` | `0x04-0x05` | `0x84-0x85` | X-axis cycle count |
| `CCY` | `0x06-0x07` | `0x86-0x87` | Y-axis cycle count |
| `CCZ` | `0x08-0x09` | `0x88-0x89` | Z-axis cycle count |
| `TMRC` | `0x0B` | `0x8B` | Continuous-mode update rate |
| `MX` | `0x24-0x26` | `0xA4-0xA6` | X-axis measurement result |
| `MY` | `0x27-0x29` | `0xA7-0xA9` | Y-axis measurement result |
| `MZ` | `0x2A-0x2C` | `0xAA-0xAC` | Z-axis measurement result |
| `BIST` | `0x33` | `0xB3` | Built-in oscillator self-test |
| `STATUS` | `0x34` | `0xB4` | Data-ready status in bit 7 |
| `HSHAKE` | `0x35` | `0xB5` | DRDY clear behavior and NACK diagnosis |
| `REVID` | `0x36` | `0xB6` | Revision ID |

Key default values:

- Cycle count defaults to `0x00C8` on each axis, which is **200** decimal.
- `TMRC` default is `0x96`, about **37 Hz** in continuous mode.
- `HSHAKE` default is `0x1B`.

## Single Measurement

### Recommended bring-up sequence

Use this sequence to get a first 3-axis reading:

1. Power the device and confirm SPI wiring.
2. Optionally read `REVID` from `0x36` to confirm communication.
3. Optionally write cycle counts if you do not want the default `200`.
4. Start a 3-axis single measurement by writing `POLL = 0x70`.
5. Wait for `DRDY` to go HIGH, or poll bit 7 of `STATUS`.
6. Read 9 bytes starting with read command `0xA4`.
7. Convert the three 24-bit signed results into `X`, `Y`, and `Z`.

### Useful command values

- Single 3-axis measurement: `POLL = 0x70`
- Default cycle count per axis: `200 = 0x00C8`
- Faster example cycle count per axis: `100 = 0x0064`

### C-like SPI helper functions

These examples assume your platform already has a low-level SPI transfer routine and GPIO control for `SSN`.

```c
#include <stdint.h>
#include <stdbool.h>

static void rm3100_select(void);
static void rm3100_deselect(void);
static void spi_txrx(const uint8_t *tx, uint8_t *rx, uint32_t len);
static bool rm3100_drdy_is_high(void);

void rm3100_write_reg(uint8_t reg, uint8_t value)
{
    uint8_t tx[2] = { reg, value };
    rm3100_select();
    spi_txrx(tx, 0, 2);
    rm3100_deselect();
}

void rm3100_write_regs(uint8_t start_reg, const uint8_t *data, uint32_t len)
{
    rm3100_select();
    spi_txrx(&start_reg, 0, 1);
    spi_txrx(data, 0, len);
    rm3100_deselect();
}

void rm3100_read_regs(uint8_t start_reg, uint8_t *data, uint32_t len)
{
    uint8_t cmd = 0x80u | start_reg;
    rm3100_select();
    spi_txrx(&cmd, 0, 1);
    spi_txrx(0, data, len);
    rm3100_deselect();
}
```

### Programming cycle counts

Cycle count controls the tradeoff between resolution, power, and speed:

- Higher cycle count: more gain and better resolution, but slower
- Lower cycle count: faster measurements and lower power, but lower resolution

The simplest way to program all three axes at once is to write 6 bytes starting at `0x04`.

Example: set all axes to **100** cycle counts (`0x0064`):

```c
void rm3100_set_cycle_count_xyz(uint16_t cc)
{
    uint8_t buf[6] = {
        (uint8_t)(cc >> 8), (uint8_t)(cc & 0xFF),
        (uint8_t)(cc >> 8), (uint8_t)(cc & 0xFF),
        (uint8_t)(cc >> 8), (uint8_t)(cc & 0xFF)
    };

    rm3100_write_regs(0x04, buf, sizeof(buf));
}
```

### Reading one 3-axis sample

The measurement registers hold signed 24-bit 2's-complement values. Read all 9 bytes in one burst starting at `0x24`.

```c
static int32_t rm3100_sign_extend_24(uint32_t raw)
{
    if (raw & 0x00800000u) {
        raw |= 0xFF000000u;
    }
    return (int32_t)raw;
}

bool rm3100_read_xyz(int32_t *x, int32_t *y, int32_t *z)
{
    uint8_t raw[9];

    rm3100_read_regs(0x24, raw, sizeof(raw));

    *x = rm3100_sign_extend_24(
        ((uint32_t)raw[0] << 16) |
        ((uint32_t)raw[1] << 8)  |
        ((uint32_t)raw[2] << 0));

    *y = rm3100_sign_extend_24(
        ((uint32_t)raw[3] << 16) |
        ((uint32_t)raw[4] << 8)  |
        ((uint32_t)raw[5] << 0));

    *z = rm3100_sign_extend_24(
        ((uint32_t)raw[6] << 16) |
        ((uint32_t)raw[7] << 8)  |
        ((uint32_t)raw[8] << 0));

    return true;
}
```

### Complete single-shot example

```c
bool rm3100_take_single_measurement(int32_t *x, int32_t *y, int32_t *z)
{
    uint8_t status;

    rm3100_set_cycle_count_xyz(200);   // Datasheet default, good resolution
    rm3100_write_reg(0x00, 0x70);      // POLL = measure X, Y, Z once

    while (!rm3100_drdy_is_high()) {
        // Or poll STATUS if DRDY is not connected
    }

    rm3100_read_regs(0x34, &status, 1); // Optional STATUS check
    if ((status & 0x80u) == 0) {
        return false;
    }

    return rm3100_read_xyz(x, y, z);
}
```

If `DRDY` is not wired, poll `STATUS` until bit 7 becomes `1`:

```c
bool rm3100_wait_for_data_ready(void)
{
    uint8_t status;

    do {
        rm3100_read_regs(0x34, &status, 1);
    } while ((status & 0x80u) == 0);

    return true;
}
```

## Continuous Measurement

Continuous mode is useful when you want the sensor to keep sampling in the background.

### How it works

1. Write `TMRC` to choose the update rate.
2. Write `CMM` to select axes and start continuous mode.
3. Wait for `DRDY`.
4. Read 9 bytes starting at `0x24`.
5. Repeat.

For the common 3-axis case:

- `CMM = 0x79` means:
  - Start continuous mode
  - Measure X, Y, and Z
  - Set `DRDY` after the full 3-axis sequence completes
  - Alarm mode disabled in normal use

Selected `TMRC` values from the datasheet:

| `TMRC` | Approx. Rate |
| --- | --- |
| `0x92` | 600 Hz |
| `0x93` | 300 Hz |
| `0x94` | 150 Hz |
| `0x95` | 75 Hz |
| `0x96` | 37 Hz |
| `0x97` | 18 Hz |
| `0x98` | 9 Hz |
| `0x99` | 4.5 Hz |
| `0x9A` | 2.3 Hz |

The actual top rate is also limited by cycle count. For example, a 3-axis cycle count of 200 limits the maximum 3-axis rate to about 430 Hz even if `TMRC` requests more.

### Continuous mode example

```c
void rm3100_start_continuous_mode(uint8_t tmrc)
{
    rm3100_write_reg(0x0B, tmrc);  // TMRC
    rm3100_write_reg(0x01, 0x79);  // CMM: start, XYZ enabled, DRDY on full sequence
}

void rm3100_stop_continuous_mode(void)
{
    rm3100_write_reg(0x01, 0x00);
}
```

Important behavior notes:

- Reading the measurement result registers clears `DRDY` by default.
- Any register write also clears `DRDY` by default.
- Reading `CMM` or writing `TMRC` can terminate continuous mode, so do not do that casually while streaming data.

## Data Conversion

The RM3100 returns raw signed **24-bit 2's-complement** values.

Raw output is not automatically in `uT`; you must divide by the gain that matches your cycle count.

Datasheet gain values for common cycle counts:

| Cycle Count | Gain |
| --- | --- |
| `50` | `20 LSB/uT` |
| `100` | `38 LSB/uT` |
| `200` | `75 LSB/uT` |

Conversion formula:

```c
float field_uT = raw_counts / gain_lsb_per_uT;
```

Example:

```c
float rm3100_counts_to_uT(int32_t raw_counts, float gain_lsb_per_uT)
{
    return (float)raw_counts / gain_lsb_per_uT;
}
```

Notes:

- Use the gain that matches the cycle count you programmed.
- For cycle counts not listed in the datasheet table, you may need calibration or interpolation.
- Earth field is roughly `50 uT`, so this is a good sanity check during first bring-up.

## Troubleshooting

### `DRDY` never goes HIGH

Check:

- `I2CEN` is really LOW for SPI
- `SSN` is asserted correctly for each transaction
- SPI mode is 0 or 3
- SPI clock is not above 1 MHz
- The `POLL` write really sent `0x00, 0x70`
- The device is powered correctly and `AVDD` and `DVDD` are valid
- `REXT` is populated correctly

If `DRDY` is not wired, poll `STATUS` register `0x34` and watch bit 7.

### Reading before data is ready

If you read measurement registers before data is ready, the `HSHAKE` register can report a NACK reason:

- `NACK2 = 1`: measurement result registers were read while `DRDY = 0`

### Writing `POLL` while continuous mode is active

The datasheet says a single-measurement command is ignored if continuous measurement mode is already running.

`HSHAKE` can report:

- `NACK1 = 1`: wrote `POLL` while `CMM` was in progress, or wrote `CMM` while a poll measurement was in progress

### `DRDY` seems to clear unexpectedly

The default `HSHAKE = 0x1B` enables both of these behaviors:

- `DRC0 = 1`: `DRDY` clears on any register write
- `DRC1 = 1`: `DRDY` clears when measurement results are read

This means you should read data only after it is ready, and avoid unnecessary writes between `DRDY` going HIGH and reading `MX/MY/MZ`.

### Optional self-test

`BIST` can verify the internal LR oscillator.

Typical flow:

1. Write `BIST` with `STE=1` and your chosen timeout/period bits.
2. Trigger a `POLL` command.
3. Wait for `DRDY`.
4. Read `BIST` with command `0xB3`.
5. Check `XOK`, `YOK`, and `ZOK`.

## Brief I2C Notes

This guide focuses on SPI, but the same register map also works over I2C.

I2C differences:

- Pull `I2CEN` **HIGH**
- The 7-bit slave address format is `0b01000xx`
- The lower two address bits come from the address pins
- Write transactions send the register address first, then payload bytes
- Read transactions first write the register address, then issue an I2C read
- Register auto-increment still applies, so burst reads and writes are preferred

The same key register addresses still apply:

- `POLL = 0x00`
- `CMM = 0x01`
- `CCX/CCY/CCZ = 0x04-0x09`
- `TMRC = 0x0B`
- `MX/MY/MZ = 0x24-0x2C`
- `BIST = 0x33`
- `STATUS = 0x34`
- `HSHAKE = 0x35`
- `REVID = 0x36`

## Quick Reference

For a first successful SPI bring-up:

1. Wire SPI, `DRDY`, power, and `REXT`.
2. Set `I2CEN=LOW`.
3. Optionally read `REVID` with `0xB6`.
4. Optionally write cycle counts at `0x04-0x09`.
5. Write `POLL=0x70`.
6. Wait for `DRDY` or `STATUS[7]`.
7. Read 9 bytes from `0xA4`.
8. Sign-extend each 24-bit result.
9. Convert counts to `uT` using the correct gain.

That sequence is enough to build a minimal RM3100 driver on almost any MCU or embedded host.
